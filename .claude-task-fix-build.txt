修复 SOMNI 项目 (D:/projects/somni-sleep) 的构建错误。有 3 个问题：

## 问题 1: 缺少 lucide-react 包
运行: cd D:/projects/somni-sleep && npm install lucide-react

## 问题 2: import 路径错误
NOCTURNE 项目把组件放在子目录（如 components/layout/header.tsx），但 SOMNI 项目把组件直接放在 components/ 根目录（如 components/header.tsx）。

在 D:/projects/somni-sleep/src/app/[locale]/layout.tsx 中：
- 找到 `from "@/components/layout/header"` 改成 `from "@/components/header"`
- 找到 `from "@/components/layout/footer"` 改成 `from "@/components/footer"`

## 问题 3: 全局搜索其他类似的路径不匹配
在 D:/projects/somni-sleep/src/ 下搜索所有包含 `@/components/layout/` 和 `@/components/product/` 和 `@/components/sections/` 和 `@/components/auth/` 和 `@/components/ui/` 的文件。

对于每个匹配：
- `@/components/layout/X` → `@/components/X` (把文件移到根目录)
- `@/components/product/X` → `@/components/X`
- `@/components/sections/X` → `@/components/X`
- `@/components/auth/X` → `@/components/X`
- `@/components/ui/X` → `@/components/X`

同时，如果在 SOMNI 的 src/components/ 根目录还没有对应的文件，从 D:/projects/nocturne/src/components/ 对应子目录复制过来。

完成后运行: cd D:/projects/somni-sleep && npx next build --webpack 2>&1 | tail -20
如果还有错误继续修，直到编译通过。
